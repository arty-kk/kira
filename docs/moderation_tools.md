# moderation_tools.md

Бриф для менеджера: **какие фичи модерации есть как продукт**, и **что можно включить/выключить**.


Версия для заполнения (без бинарных файлов): `docs/moderation_tools_fillable.md`.

---

## Как пользоваться этим документом

1. В колонке **«Решение (ON/OFF)»** поставьте нужное значение.
2. При необходимости добавьте короткий комментарий в колонке **«Примечание»**.
3. Передайте заполненный бриф — по нему будет собран итоговый конфиг.

## 3) Форварды, внешние сообщения, кнопки

| Фича | Что это даёт в продукте | Решение (ON/OFF) | Примечание |
|---|---|---|---|
| Удалять форварды у новичков | Новичок не может пересылать контент |  |  |
| Удалять форварды из каналов/ботов | Снижаем канал-спам и автопосты |  |  |
| Удалять внешние сообщения каналов | В чат не попадают посты из внешних каналов |  |  |
| Удалять сообщения с кнопками | Блокируем промо/кнопочные объявления |  |  |
| Удалять inline-bot сообщения | Снижаем автоматический бот-контент |  |  |

Примечания по типам пересылок и каналов:
- `is_automatic_forward` (автоперенос из связанного канала в комментарии) не подпадает под флаг «Удалять форварды из каналов/ботов».
- Обычные forwards (`forward_from`, `forward_from_chat`, `forward_sender_name`) регулируются фичей «Удалять форварды из каналов/ботов».
- Сообщения с источником linked-channel (`from_linked`) не удаляются правилами для внешних каналов/форвардов.
- trusted-repost (перепост из доверенного источника в доверенный контур) исключается из авто-удаления форвардов/канальных сообщений.

## 7) Что делать с нарушениями

| Фича | Что это даёт в продукте | Решение (ON/OFF) | Примечание |
|---|---|---|---|
| Удалять сообщения, помеченные AI как blocked | Автоматическая реакция без ожидания модератора для самых тяжёлых кейсов |  |  |
| Удалять сообщения, помеченные AI как flagged | Автоматическая реакция на спорные AI-флаги до ручной проверки |  |  |
| Deep AI учитывает историю диалога | Если ON — в deep-проверку добавляется контекст истории + `NEW MESSAGE`; если OFF — проверяется только текущее сообщение |  | Default: OFF |
| Показывать алерты модераторам | Ручной контроль спорных кейсов |  |  |
| Разрешить модераторам бан из алерта | Быстрое ручное действие в 1 клик |  |  |
| При бане удалять сообщения нарушителя | Убираем «следы» спама после бана |  |  |

---


### Пример `.env` для deep-модерации

```env
# Консервативный режим: deep-проверка только текущего сообщения
MODERATION_DEEP_INCLUDE_HISTORY=false

# Режим с контекстом: deep-проверка history + NEW MESSAGE
# MODERATION_DEEP_INCLUDE_HISTORY=true
```

## 8) Комментарии канала (comment-mode)

| Фича | Что это даёт в продукте | Решение (ON/OFF) | Примечание |
|---|---|---|---|
| Включить модерацию комментариев | Отдельные правила для связки канал ↔ чат |  |  |
| Режим ссылок `group_default` (строгий) | В комментариях действуют обычные ограничения |  |  |
| Режим ссылок `relaxed` (мягкий) | В комментариях мягче правила по ссылкам/каналам |  | Выбрать только один режим |
| Удалять внешние replies в comments | Жёстче чистим ветки комментариев |  |  |

Уточнения для comment-mode:
- Для `from_linked` (сообщение пришло из связанного канала) действует bypass на удаление как для канальных сообщений, так и для форвардов.
- `is_automatic_forward` трактуется как системный автоперенос linked-channel, а не как обычный пользовательский forward.
- Исключение trusted-repost применяется только если destination и source входят в доверенный контур (trusted scope).

## 10) Готовые пакеты (можно выбрать один)

### Пакет A — «Строго антиспам»
- ON: удалять любые ссылки, Telegram-ссылки, кнопки, форварды у новичков, бан за первую ссылку новичка.
- ON: авто-удаление blocked AI.
- Подходит для публичных чатов с высоким потоком спама.

### Пакет B — «Сбалансированный»
- ON: Telegram-ссылки, кнопки, алерты модераторам.
- OFF: удалять вообще все ссылки и тотальный бан новичков.
- Подходит для комьюнити, где важна и безопасность, и нормальное общение.

### Пакет C — «Мягкий + ручной контроль»
- ON: алерты модераторам, AI-сигналы.
- OFF: часть авто-удалений и авто-санкций.
- Подходит для чатов, где нужна осторожная модерация и ручные решения.

## 11) Дополнительные фичи для чатов (помимо модерации)

| Фича | Что это даёт в продукте | Решение (ON/OFF) | Примечание |
|---|---|---|---|
| AI-приветствие новых участников в группе | Бот приветствует новых пользователей в групповом чате |  |  |
| Автоответ в группе по теме обсуждения | Бот может отвечать не только по прямому упоминанию, но и по релевантной теме |  |  |
| Ответы бота только по упоминанию | Бот отвечает только при явном триггере (упоминание/команда), без автоответа «в фоне» |  |  |
| Дневной лимит ответов бота в группе | Ограничивает активность бота в группе за сутки |  | Указать лимит |
| Команда `/battle` в группе | Включает игровой/интерактивный сценарий по команде |  |  |
| Работа в комментариях канала (канал ↔ чат) | Бот и правила доступны в связке комментариев канала |  |  |
