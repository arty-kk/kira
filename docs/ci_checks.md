# CI checks

В CI пайплайн запускает `bash scripts/check.sh` после установки зависимостей и инструментов проверки.

## Обязательные проверки

`bash scripts/check.sh` всегда выполняет:
- `pytest -q`
- `python -m compileall app`

Условные проверки:
- `ruff check .` — запускается только если есть один из конфигов: `pyproject.toml`, `ruff.toml`, `.ruff.toml`.
- `pyright` — запускается только если есть `pyrightconfig.json`.

## Поведение при отсутствии бинарей

- Локально (не-CI): если для условной проверки найден конфиг, но бинарь инструмента отсутствует, проверка пропускается с явным сообщением.
- В CI (`CI=true`): если найден конфиг условной проверки, но бинарь отсутствует, `scripts/check.sh` завершается с ошибкой (ненулевой код), и пайплайн падает.

Это гарантирует, что ожидаемые проверки не «теряются» из-за отсутствующего инструмента в окружении CI.
